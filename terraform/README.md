# Security Collaboration Terraform (IaC)

본 저장소는 클라우드 환경의 애플리케이션 보안, 인프라 보안 및 권한 통제를 위한 인프라스트럭처 에즈 코드(IaC) 구성을 관리합니다.

## 🛡️ 최고 수준의 보안 통제: Production 환경 구성
현재 디렉토리(`environments/prod/iam-policy/`)의 작업 내역은 `[dev, staging, prod]` 환경 중 **`prod` (프로덕션)** 환경을 타겟으로 합니다. 
운영(Production) 환경은 단 한 번의 설정 오류나 인가되지 않은 접근이 치명적인 장애로 직결될 수 있으므로, **RBAC(역할 기반 접근 제어)** 및 **SoD(직무 분리/상호 견제)** 원칙을 엄격하게 적용하여 **가장 빡빡한(Strict) 수준의 보안 아키텍처**를 기준점(Baseline)으로 수립했습니다.

## 🧩 Terraform 파일별 역할 및 유기적 구조 (How it works)
이 구조는 단순히 권한을 나열한 것이 아니라, **"작은 권한 조각(블록)들을 조립하여 직무(Role)를 만들고, 사용자가 그 역할을 임시로 입고(Assume) 일하는 구조"**로 유기적으로 연결되어 있습니다.

1. **`variables.tf` / `main.tf`**: (기반 설정)
   - 프로젝트 이름, 배포 환경(예: prod), AWS 리전 등의 기초 변수를 정의하고 Terraform 프로바이더를 설정합니다.
2. **`iam_labels.tf`**: (레고 블록)
   - 보안 정책의 가장 작은 단위(예: '읽기 전용', '인프라 생성', '시크릿 관리')인 **14개의 표준 보안 권한 모듈**을 정의합니다.
3. **`iam_policies.tf`**: (권한 조립)
   - `iam_labels.tf`에서 만든 레고 블록들을 각 직무에 맞게 조합하여, 실제 AWS 상에 **11개 직무별 코어 정책(Policy)** 리소스를 생성합니다.
4. **`iam_roles.tf`**: (옷장 및 역할 생성)
   - 사용자가 갈아입을 수 있는 11벌의 **직무 역할(Role)**을 만들고, 앞서 조립한 코어 정책을 각 역할에 입혀둡니다. 이때 '옷을 입을 수 있는(Assume) 조건'으로 **MFA(다중 인증) 필수**라는 강력한 자물쇠(Trust Relationship)를 채워둡니다.
5. **`iam_users_and_groups.tf`**: (사용자 배정 및 출입증 발급)
   - 실제 팀원들의 IAM 계정을 만들고 그룹에 배정합니다. **팀원들은 평소에는 아무런 실무 권한이 없으며**, 오직 자신이 배정된 그룹에 허락된 특정 역할(Role)로 **스위칭(Assume)할 수 있는 권한**만 부여받습니다.

## 🏗️ 아키텍처 원칙 (Architecture Principles)

- **RBAC (역할 기반 접근 제어)**
  - 개별 사용자에게 파편화된 정책을 주지 않고, 철저히 사전에 정의된 '직무(Role)' 단위로만 권한을 부여하고 통제합니다.
- **SoD (직무 분리 및 상호 견제)**
  - 개발자는 서버 메인 구조를 바꿀 수 없고, 인프라 담당자는 보안 통제를 스스로 해제할 수 없으며, 보안 담당자는 인프라 생성이 불가하도록 권한을 쪼개어 내부 위협(Insider Threat)을 원천 차단합니다.
- **AssumeRole (임시 권한 위임 체계)**
  - 모든 작업자의 상시 코어 권한은 **Zero(0)**입니다. 인프라 작업이 필요할 때만 MFA 인증을 거쳐 임시(Temporary) 자격 증명(AssumeRole)을 획득하며, 자격 증명이 탈취되더라도 만료 시간이 짦아 피해가 극히 제한적입니다.
- **DRY & Scalability (중복 제거 및 확장성)**
  - `for_each` 반복문과 배열 매핑을 적극 활용하여 코드를 단일 진실 공급원(Single Source of Truth)으로 구축했습니다. 새 인원이 합류하더라도 변수에 이름 한 줄만 추가하면 모든 인프라 권한 매핑이 리소스 복제 없이 자동 반영됩니다.

> ⚠ **주의**: Terraform State 파일(`terraform.tfstate`) 및 `terraform.tfvars` 등 민감 정보는 절대 외부에 커밋하지 않습니다.
